2Quick start — run the whole suite

The orchestrator seeds the 200-user cohort + every bucket's preconditions, then runs the journey-flow walk and all 10 bucket tests serially, writing screenshots + regenerating RESULTS.md. Run it from the e2e/ directory:

cd e2e

# seed everything, then run everything (default):
node flutter-suite/run-suite.cjs --all

# or split the two phases:
node flutter-suite/run-suite.cjs --seed    # (re)seed cohort + all bucket preconditions
node flutter-suite/run-suite.cjs --run     # run all tests (assumes already seeded)

# run just a subset:
ONLY=queue,forms,journey node flutter-suite/run-suite.cjs --run
Evidence lands in breakthroughs-flutter/mobile-evidence/<bucket>/ — one folder per bucket, files named in drive order (<bucket>-<step>-<what>.png). Browse with open breakthroughs-flutter/mobile-evidence then select-all + Space for a Quick Look slideshow.

3Run a single bucket (the fast iterate loop)

Each run re-seeds first (the app mutates its data), then drives one test. Run from e2e/flutter-suite/. Substitute <bucket> and the user <idx> from the table below:

cd e2e/flutter-suite

node seed-<bucket>.cjs --seed \
  && SKIP_PUBGET=1 \
     TEST_TARGET=integration_test/features/<bucket>_test.dart \
     E2E_EMAIL=participant<idx>+jrny@example.com \
     E2E_LABEL=<bucket> \
     E2E_EVIDENCE=<bucket> \
     node run-flutter-test.cjs

Concrete example — the journey bucket (user 92)

cd e2e/flutter-suite
node seed-journey.cjs --seed \
  && SKIP_PUBGET=1 TEST_TARGET=integration_test/features/journey_test.dart \
     E2E_EMAIL=participant92+jrny@example.com E2E_LABEL=journey E2E_EVIDENCE=journey \
     node run-flutter-test.cjs

4Firestore composite indexes

Some screens need composite indexes on the test project. They're declared in firestore.indexes.json and deployed with a dedicated config:

firebase deploy --only firestore:indexes \
  --config firebase.indexes.json \
  --project slabs-queue-e2e-exdcz \
  --non-interactive
Footgun: the main firebase.json is hosting-only and points at PRODUCTION. ALWAYS pass --config firebase.indexes.json --project slabs-queue-e2e-exdcz. Index builds take 1–5 min; a run started too early fails with "index currently building" — wait and re-run.

5Gotchas (read before debugging a red run)

Always flutter analyze the test before running. A compile error makes flutter drive silently run the LAST successful build (you'll debug the wrong test). 
cd breakthroughs-flutter && /opt/homebrew/bin/flutter analyze integration_test/features/<bucket>_test.dart
  • Re-seed before every run. The app writes to Firestore during the test, so each run must reset its preconditions — the per-bucket command above does node seed-<bucket>.cjs --seed && … for you.
  • Read the FULL log, not the tail. The runner only prints the last 30 lines on failure; the complete output is at $TMPDIR/flutterdrive-<label>.log — grep it for the real cause (requires an index, was thrown, your CONTENT/AUTH/…[label]: prints).
  • breakthroughs-flutter is its own nested git repo. Commit test files there (git -C breakthroughs-flutter add … && commit); seeds / indexes / RESULTS commit to the parent repo.
  • One Flutter build at a time. Don't run two flutter drives against this repo's build/ios concurrently — they corrupt each other. If a run shows output from a different test, kill the other session and flutter clean once.
  • Sim-blocked legs are honest. Real-media players (BetterPlayer/just_audio), CallKit/push, camera/QR, OpenAI and external links can't run headless — those legs are drive-to-screen + render-asserted and labelled in RESULTS.md, never faked green.

6What "green" means

  • Runner prints ✓ <bucket> PASSED · N screenshots and exits 0; the flutter log ends with All tests passed.
  • Every functional assertion is anti-circular: it polls the server for the doc the app wrote (e.g. profile_data.last_login, clientissue, arenalayers.attended), never a value the seed wrote.
  • A trailing +1 -1 … Some tests failed with exit 0 + All tests passed. is benign post-test (tearDownAll) noise — the runner's ✓ PASSED verdict is authoritative.
